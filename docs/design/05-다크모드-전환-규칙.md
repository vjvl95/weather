# 05 — 다크모드 전환 규칙

> 다크모드 지원 구현 시 준수해야 할 규칙을 정의합니다.

---

## colorScheme 옵션

| 값 | 동작 |
|----|------|
| `'system'` | 기기 시스템 설정을 따름 (기본값) |
| `'light'` | 항상 라이트 모드 |
| `'dark'` | 항상 다크 모드 |

---

## 구현 규칙

### 1. 네이티브 UI 동기화
```typescript
import { Appearance } from 'react-native';

// 네이티브 키보드/Alert 등도 다크모드 반영
Appearance.setColorScheme(resolvedColorScheme);
```

### 2. 기존 사용자 방어 코드
```typescript
// AsyncStorage에서 복원할 때 필드가 없을 수 있음
const colorScheme = stored.colorScheme ?? 'system';
```

### 3. 테마 토큰 dark 지원
```typescript
// registerTheme 시 light + dark 토큰 세트 제공
registerTheme('mytheme', {
  light: LIGHT_TOKENS,
  dark: DARK_TOKENS,
});
```

---

## 하드코딩 색상 금지

```typescript
// ❌ 하드코딩 금지
<View className="bg-white text-black" />

// ✅ 테마 토큰 사용
const { colors } = useTheme();
<View style={{ backgroundColor: colors.background }}>
```
