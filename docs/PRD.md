# PRD: 몽글날씨 (가칭)

> 지친 일상에서 귀여움을 무기로 리텐션을 끌어올리는 날씨 앱

---

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| **한 줄 정의** | 귀여운 구름 슬라임 캐릭터 "몽글이"가 날씨를 알려주는 앱 |
| **타겟 유저** | 중고등학생 ~ 2030 남녀 |
| **분위기** | 귀엽고 아기자기한 |
| **레퍼런스** | Callie (캐릭터 유대감), 하루날씨 (날씨 캐릭터 표현) |
| **플랫폼** | Android / iOS 동시 출시 |
| **수익 모델** | v1 무료 / v2+ 광고 검토 |
| **앱 이름** | 추후 결정 (가칭: 몽글날씨) |

---

## 2. 캐릭터: 몽글이

### 2-1. 기본 설정

| 항목 | 내용 |
|------|------|
| **이름** | 몽글이 |
| **형태** | 구름 슬라임 (메이플 슬라임 스타일의 뭉실한 구름 판타지 생물) |
| **성격** | 친절하고 다정한 존댓말 ("오늘 비가 와요! 우산 챙기세요~") |
| **캐릭터 수** | v1: 1개 고정 / v2+: 여러 캐릭터 추가 |

### 2-2. 날씨별 변화 요소 (v1)

| 요소 | v1 | v2+ |
|------|-----|------|
| 표정 변화 | ✅ | ✅ |
| 색상 변화 | ✅ | ✅ |
| 배경 변화 | ✅ | ✅ |
| 소품 (우산, 목도리 등) | | ✅ |
| 꾸미기 (옷, 악세사리) | | ✅ |

### 2-3. v1 날씨 상태 (5종)

| 날씨 | 몽글이 표정 | 몽글이 색상 | 배경 |
|------|-------------|-------------|------|
| ☀️ 맑음 | 행복, 방긋 | 하얗고 밝은 | 밝은 하늘색 |
| ☁️ 흐림 | 무표정, 졸림 | 연한 회색 | 흐린 회색 하늘 |
| 🌧️ 비 | 찡그림, 살짝 슬픔 | 진한 회색 | 어두운 하늘 + 빗방울 |
| ❄️ 눈 | 신남, 반짝 | 연한 파란빛 | 하얀 눈 내리는 하늘 |
| 🌙 밤 | 나른, 졸림 | 연한 남색 | 어두운 밤하늘 + 별 |

> 확장 가능한 구조로 설계하여 천둥번개, 안개, 강풍, 폭염, 한파 등 추후 쉽게 추가

### 2-4. 교감 요소 (v1)

| 기능 | 설명 |
|------|------|
| **날씨 반응** | 날씨 상태에 따라 표정/색상/배경 자동 변화 |
| **터치 반응** | 탭하면 통통 튀기, 쓰다듬으면 좋아하는 반응 |
| **말풍선** | 날씨에 맞는 한마디 ("오늘 비가 와요! 우산 챙기세요~") |

## 3. 기능 범위

### 3-1. v1 기능

| 기능 | 설명 |
|------|------|
| **현재 날씨** | 온도, 날씨 상태, 체감온도 등 |
| **시간별 예보** | 3시간 단위 예보 |
| **주간 예보** | 7일간 예보 |

| **도시 검색** | 원하는 도시 검색하여 날씨 확인 |
| **캐릭터 몽글이** | 날씨별 표정/색상/배경 변화 + 터치 반응 + 말풍선 |

### 3-2. v2+ 로드맵

| 기능 | 우선순위 |
|------|----------|
| 캐릭터 소품 (우산, 목도리 등) | 높음 |
| 캐릭터 꾸미기 (옷, 악세사리) | 높음 |
| 다크모드 | 높음 |
| 미세먼지 / 자외선 정보 | 중간 |
| 푸시 알림 (아침 날씨, 비/눈 알림) | 중간 |
| 홈 위젯 | 중간 |
| 추가 캐릭터 | 중간 |
| 광고 | 낮음 |
| 해외 날씨 (OpenWeatherMap) | 낮음 |
| 추가 날씨 상태 (천둥번개, 안개 등) | 낮음 |

## 4. 화면 구성

### 4-1. 탭 구조 (하단 탭바)

```
[ 홈 ] [ 시간별 ] [ 주간 ] [ 설정 ]
```

### 4-2. 화면별 명세

| 화면 | 주요 요소 |
|------|-----------|
| **홈** | 몽글이 캐릭터 (크게) + 현재 온도/날씨 상태 + 상단 위치명 (탭하면 도시 검색) |
| **시간별** | 시간별 예보 리스트 (온도, 날씨 아이콘) |
| **주간** | 7일 예보 리스트 (최저/최고 온도, 날씨 아이콘) |
| **설정** | 위치 관리 (도시 검색/추가), 기본 설정 |
| **도시 검색** | 도시명 입력 → 검색 결과 → 선택 (홈 상단 + 설정에서 접근) |

### 4-3. 온보딩

- 별도 온보딩 없음
- 첫 실행 시 위치 권한만 간단히 요청

## 6. 기술 스택

| 항목 | 기술 |
|------|------|

## 7. 설계 원칙

1. **핵심 먼저, 확장 가능하게** — v1은 최소 기능, 구조는 v2+ 대비
2. **캐릭터가 주인공** — 날씨 정보보다 몽글이가 먼저 눈에 들어와야 함
3. **FSD 레이어 규칙 준수** — import 방향, 슬라이스 구조, public API 원칙
4. **날씨 상태 확장 용이** — 새 날씨 추가 시 매핑 데이터만 추가하면 되는 구조
